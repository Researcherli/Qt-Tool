# 1. 项目优化背景

当前软件已经具备基础桌面工具框架，但现有界面与功能规划存在以下问题：

1. 当前 UI 风格偏默认浅色灰白，整体观感普通，不符合新的审美方向。
2. 功能入口较多，CAN、协议解析、设置、插件等内容会让当前版本显得复杂。
3. 串口页面功能堆叠较明显，接收区、发送区、快捷命令区的视觉层级不够清楚。
4. 首页功能不够明确，需要从“展示页”变成“工作入口页”。
5. 当前阶段真正需要的功能应聚焦在：
   - 串口通信调试
   - 数据格式转换
   - BIN 文件导入、解析、字符串提取、搜索定位

因此，本次优化的核心思路是：

> 做减法，聚焦核心工具；做重构，让界面更清晰；做风格统一，让软件拥有木质古朴、稳重耐看的整体观感。

## 5.5 控件风格原则

### 按钮

- 主按钮使用深棕色背景。
- 次按钮使用浅棕背景和深棕文字。
- 危险操作使用红棕色。
- 按钮不要过圆，保持工具软件的稳重感。

### 输入框

- 使用浅米色背景。
- 聚焦时边框变为老铜色。
- 输入框文字使用深棕色。
- 数据输入框使用等宽字体。

### 表格

- 表头使用淡木色背景。
- 行之间有轻微分隔线。
- 选中行使用浅铜色高亮。
- 表格内容尽量左对齐。

### 数据显示区

- 必须使用等宽字体。
- 背景略浅于主背景。
- HEX、ASCII、偏移地址分列展示。
- 数据密集区域不使用强阴影。

# 8. 串口工具页面 UI 结构设计

## 8.1 串口工具页面目标

串口工具页面是当前版本的第一核心功能，目标如下：

1. 串口配置清晰。
2. 连接状态明确。
3. 接收区足够大，数据显示清楚。
4. 发送区独立，操作方便。
5. 快捷命令区规整，不再像简单列表。
6. 支持文本和 HEX 两种收发模式。
7. 支持循环发送、追加换行、时间戳、日志保存。

# 附录 E：BIN 字符串提取规则建议

默认提取规则：

   - `0x20 ~ 0x7E`
   - 可选包含 `\r`、`\n`、`\t`
3. 提取结果记录：
   - 起始偏移
   - 长度
   - 内容
   - 编码类型
4. 点击结果后跳转到 HEX 视图对应偏移。

# 附录 F：HEX 输入解析规则建议

允许以下格式：

```text
4A 43 58 58
4A435858
0x4A 0x43 0x58 0x58
4A,43,58,58
```

解析时统一转换为：

```text
4A 43 58 58
```

非法情况：

| 输入 | 错误原因 |
|---|---|
| `4A 43 5` | HEX 位数不完整 |
| `4A GG 58` | 包含非法字符 |
| `0x4A 0x` | 前缀后无数据 |
| `10102` | 二进制中出现非 0/1 字符 |
