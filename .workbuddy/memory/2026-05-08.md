# 2026-05-08

## 修复 est_app.exe 间歇性崩溃

### 背景
est_app.exe 启动时间歇性崩溃（约 62% 崩溃率），表现为启动后 2-55 秒内 "terminated abnormally"。

### 定位的根因
1. **QSS 引用不存在的资源** (`wood_classic.qss`, `industrial_dark.qss`) 中 `QCheckBox::indicator:checked` 引用了 `url(:/icons/check.png)`，该资源不在 QRC 中，可能导致渲染时不可预测行为
2. **SerialAssistantPage 构造时直接调用 QSerialPortInfo::availablePorts()** — Windows 串口枚举在驱动状态转换时可能阻塞或崩溃
3. **MemoryAnalyzerWidget 使用临时 RecentRecordManager 对象** — `loadElfFile()`, `loadMapFile()`, `refreshRecentFiles()` 三处使用 `RecentRecordManager(this)` 创建临时对象，导致不必要的文件 I/O
4. **Qt 6.11 缺少字体目录** — 日志显示 `QFontDatabase: Cannot find font directory`

### 修改的文件
- `src/resources/themes/wood_classic.qss` — 移除 `image: url(:/icons/check.png)`
- `src/resources/themes/industrial_dark.qss` — 同上
- `src/resources/qt.conf` — 新建文件，配置 Qt Prefix 路径
- `src/app/CMakeLists.txt` — 添加 qt.conf 复制到构建输出目录
- `src/app/pages/SerialAssistantPage.cpp` — 延迟 `refreshPorts()` 到事件循环就绪后
- `src/app/widgets/MemoryAnalyzerWidget.h` — 添加 `RecentRecordManager*` 成员和 setter
- `src/app/widgets/MemoryAnalyzerWidget.cpp` — 改用成员指针代替临时对象
- `src/app/pages/MemoryAnalyzerPage.h` — 添加 `setRecentRecordManager` 接口
- `src/app/pages/MemoryAnalyzerPage.cpp` — 实现接口并透传到 widget
- `src/app/appshell.cpp` — 为 MemoryAnalyzerPage 设置 RecentRecordManager

## 修复 DiffMinimapWidget 缩略图差异显示不全和拖动跳转

### Bug 1：差异显示不全
**原因**：`rebuildCache()` 使用 run-batching（按相同状态聚合色块），当文件行数远大于缩略图高度（hRatio < 1）时每个 run 只有 1 像素，差异块不可见。
**修复**：在 hRatio < 1.5 时切换为像素级差异优先绘制，逐像素行扫描对应源行范围，有差异则优先绘制差异颜色。

### Bug 2：不能拖动跳转
**原因**：只实现了 mousePressEvent（点击跳转），缺少拖动支持。
**修复**：添加 mouseMoveEvent（拖动中持续跳转）+ mouseReleaseEvent（停止拖动），使用 qBound 防越界。

### 修改的文件
- `src/app/widgets/DiffMinimapWidget.h` — 添加 mouseMoveEvent/mouseReleaseEvent/m_isDragging
- `src/app/widgets/DiffMinimapWidget.cpp` — 实现像素级差异优先绘制 + 拖动跳转

## 全量修复偶发性闪退问题（6项）

### P0-1: 串口枚举崩溃
- **问题**: `QSerialPortInfo::availablePorts()` 在 USB-串口驱动异常时触发 SEH 异常导致进程崩溃
- **修复**: 新建 `SerialPortEnumerator.cpp/h` 独立编译单元（/EHa），用 `__try/__except` SEH 保护；`SerialConfigBar::showPopup()` 改为延迟触发 `refreshRequested`
- **文件**: `pages/SerialPortEnumerator.cpp/h`(新), `pages/SerialAssistantPage.cpp`, `widgets/SerialConfigBar.cpp`, `CMakeLists.txt`

### P0-2: 高频串口接收QTextDocument损坏
- **问题**: `SerialReceiveView::appendPacket()` 手动 `cursor.removeSelectedText()` + `deleteChar()` 删除旧行，高频时损坏 QTextDocument 内部结构
- **修复**: 用 `QPlainTextEdit::setMaximumBlockCount(5000)` 替代手动删除
- **文件**: `widgets/SerialReceiveView.cpp`

### P1-1: 插件DLL加载崩溃
- **问题**: `QPluginLoader::instance()` 加载不兼容DLL直接崩溃，无元数据预检
- **修复**: 加载前先检查 `QPluginLoader::metaData()` 的 IID 和 Qt 版本号
- **文件**: `core/plugin/PluginLoader.cpp`

### P1-2: FileComparePage渲染竞态条件
- **问题**: `renderNextChunk()` 通过 QTimer 分块渲染时，`clearComparison()` 可能清空 pending 数据
- **修复**: `renderNextChunk()` 开头增加空数据防御检查，`m_pendingLineLists` 访问增加边界检查
- **文件**: `pages/FileComparePage.cpp`

### P1-3: RecentRecordManager并发JSON读写
- **问题**: 快速串口连接/断开时并发读写 JSON 文件
- **修复**: 为所有公共方法添加 `QMutexLocker` 保护
- **文件**: `core/services/RecentRecordManager.cpp/h`

### P2: QPushButton::setMenu双重释放
- **问题**: `setMenu(menu)` 后 `delete oldMenu`，但 setMenu 已接管所有权
- **修复**: 移除 `delete oldMenu`，改用 `Q_UNUSED(oldMenu)`
- **文件**: `pages/FileComparePage.cpp`

### 技术要点
- MSVC 的 `__try/__except` 不能在含需析构局部 C++ 对象的函数中使用（C2712），需隔离到无 C++ 栈对象的函数
- `/EHa` 编译选项允许异步异常但不豁免 C2712 限制
- `setMaximumBlockCount()` 是 QPlainTextEdit 内置的行数限制方案，比手动 cursor 操作安全
