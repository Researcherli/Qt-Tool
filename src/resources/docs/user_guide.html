1. 串口工具

串口调试工具是整个平台的核心通信入口。通过串口连接目标板后，其他功能（波形图、RTOS 监控）可自动接收数据。

连接设备

  1. 将开发板通过 USB 串口线连接到电脑
  2. 在 串口设置栏 选择正确的串口号（如 COM3、COM4）
  3. 配置参数：波特率 数据位 校验位 停止位
  4. 点击 打开串口 按钮建立连接
提示：常用波特率：115200（默认）、9600（旧设备）、921600（高速）

发送数据

  • 普通发送：在输入框输入内容，点击"发送"
  • Hex 发送：切换发送模式为 HEX，输入十六进制字节（如 A5 5A 01 02）
  • 自动发送：设置定时周期，工具按间隔自动发送输入内容
  • 快捷命令：将常用指令保存为快捷命令，一键发送

接收数据

  • 显示模式：ASCII / HEX / 混合 三种视图可切换
  • 自动滚动：新数据到达时自动滚到底部
  • 搜索：使用搜索框在接收日志中定位关键字
  • 清空：一键清除当前接收缓冲区
注意：串口工具是数据源。打开波形图或 RTOS 监控页面时，数据从串口自动流向这些页面。

2. 终端控制台 NEW

终端控制台 — 类 PuTTY 风格的串口终端界面，支持 ANSI 彩色输出、命令历史、Python 脚本自动化。

功能特性

  • 终端模式：深色主题终端，串口数据实时显示，ANSI 转义码彩色支持
  • 命令历史：按 ↑/↓ 键浏览历史命令（最多保存 100 条）
  • Python 脚本：编写 Python 脚本自动化串口交互，支持发送/接收/等待/延时

终端使用

步骤 1：连接串口
在顶部连接栏选择串口号和波特率，点击 打开。
步骤 2：发送命令
在底部输入行输入命令，按回车发送（自动追加 CR+LF）。
步骤 3：浏览历史
按 ↑/↓ 键快速切换之前发送过的命令。

Python 脚本使用

切换到 脚本 标签，在代码编辑器中编写 Python 脚本：

# 可用函数：send()  send_hex()  wait()  print()  delay()
print("=== 模块自动测试 ===")
send("AT\r\n")
r = wait("OK", 3000)
if r:
    print("模块响应正常 ✓")
else:
    print("模块无响应 ✗")

点击 ▶ 运行 执行脚本，⏹ 停止 终止执行。脚本输出实时显示在右侧终端面板。

脚本 API 说明：
- send(data) — 发送字符串数据
- send_hex(hex) — 发送 HEX 数据（如 "A5 5A 03"）
- wait(pattern, timeout) — 等待串口数据匹配正则表达式（超时毫秒）
- print(msg) — 输出到终端
- delay(ms) — 延时（毫秒）

4. 协议解析 NEW

协议解析器 — 自定义二进制协议模板，将串口原始字节自动解析为结构化字段。支持帧头匹配、字段着色、校验和验证。

工作原理

协议解析器通过 模板 来描述你的二进制协议格式。创建一个模板后，当串口收到数据时，解析器自动：

  1. 在字节流中搜索 帧头（Constant 字段）
  2. 读取 长度字段（如有）确定帧长
  3. 按 字段列表 逐字段解码
  4. 验证 校验和（如有）
  5. 在帧列表中展示所有已解析的帧

创建模板

步骤 1：点击 +新建
填写模板名称（如 "UART通信协议"），选择字节序和校验算法。
步骤 2：添加字段
按帧顺序添加字段：帧头(常量)、长度、命令、数据、校验等。
步骤 3：保存模板
模板自动持久化到配置，下次启动自动加载。

支持的字段类型

类型用途示例值
常量帧头标记，用于自动匹配5A
长度帧长度字段0A
类型命令码/帧类型01
负载可变长度的数据部分01 02 03
校验XOR / 累加和 / CRC8F8
地址/序号/状态专用字段无

显示格式

每个字段可以独立选择显示格式：

  • HEX — 十六进制（默认）
  • DEC — 无符号十进制
  • 有符号 — 有符号十进制
  • BIN — 二进制
  • ASCII — ASCII 字符
  • FLOAT — 浮点数
使用场景：
• 调试自定义串口协议（如 MCU ↔ 传感器通信）
• 分析 Modbus RTU 帧（地址+功能码+数据+CRC16）
• 验证帧头 + 校验和是否正确

7. 波形图

实时波形曲线图 - 将串口数据中的数值型数据实时绘制为滚动曲线，类似于数字示波器。调试传感器、PID 控制、电机数据时的核心可视化工具。

快速上手

步骤 1：连接串口
在串口工具中打开串口，确保数据正在接收。
步骤 2：添加数据序列
切换到波形图页面，点击工具栏 添加序列 按钮。
步骤 3：配置解析规则
在弹出的对话框中选择或自定义解析规则。
步骤 4：观察实时曲线
数据自动流入图表，滚动时间窗口实时更新。

解析规则说明

波形图支持 8 种预设解析规则 + 自定义正则表达式：

规则匹配示例说明
temp=([\\d.]+)temp=25.3匹配 temp= 后面的数值
val[=:]\\s*(-?\\d+\\.?\\d*)val:123匹配 val: 或 val= 后面的数值
第一个浮点数abc 3.14 def提取文本中第一个浮点数
第一个整数x=42 ok提取文本中第一个整数
Byte 0 / Byte 10xA5 0x5A提取原始数据的第 N 个字节
Word LE原始字节流以小端序解析 16 位无符号整数
自定义正则用户定义使用正则捕获组提取数值

工具栏操作

  • 添加序列 - 弹出配置对话框，添加新的曲线
  • 删除序列 - 移除最后一条曲线
  • 暂停 / 继续 - 冻结/恢复实时更新
  • 清空 - 清除所有曲线数据
  • 时间窗口 - 调节 X 轴显示范围（1~300 秒，默认 10 秒）
使用场景示例：
- 电机控制：解析串口输出的 speed=1234，观察转速变化
- 温度采集：解析 temp=26.5，观察温升曲线
- 传感器数据：解析 adc:2048，查看 ADC 采样波形

8. RTOS 监控

RTOS 任务监视器 - 通过串口接收 FreeRTOS 的 vTaskList() 和 vTaskGetRunTimeStats() 输出，实时可视化任务状态、堆栈使用和 CPU 占比。

▎ 在你的项目中启用 FreeRTOS 调试 API

要让 EST 工具收到正确的任务数据，需要在你的 FreeRTOS 项目中做两件事：配置 FreeRTOSConfig.h 和 添加命令处理代码。

Step 1：修改 FreeRTOSConfig.h

打开你的 FreeRTOSConfig.h，确保以下宏已启用：

/* ===== FreeRTOSConfig.h 配置 ===== */

#define configUSE_TRACE_FACILITY              1
/* vTaskList() 和 vTaskGetRunTimeStats() 都需要 */

#define configGENERATE_RUN_TIME_STATS         1
/* 仅当需要 CPU 占比分析时才开启 */

#define configUSE_STATS_FORMATTING_FUNCTIONS  1
/* 启用 vTaskList() 格式化输出（FreeRTOS v10+ 需要） */

Step 2：接入 CPU 运行时间计数器

如果开启了 configGENERATE_RUN_TIME_STATS，需要实现计时器。根据你的 MCU 选择一种方式：

方案 A：使用 DWT 周期计数器（Cortex-M3/M4/M7 推荐）

/* main() 中初始化 DWT */
if (CoreDebug->DEMCR & 0x01000000) == 0 {
    CoreDebug->DEMCR |= 0x01000000;
    DWT->CYCCNT = 0;
    DWT->CTRL |= 1;
}

/* FreeRTOSConfig.h 中定义 */
#define portALT_GET_RUN_TIME_COUNTER_VALUE(x)  \
    do { x = (uint32_t)(DWT->CYCCNT); } while(0)

方案 B：使用 SysTick 或通用定时器（Cortex-M0 或其他）

/* 定义一个全局计数器，在定时器中断中累加 */
static volatile uint32_t ulRunTimeCounter = 0;

/* 定时器中断服务函数（1KHz 或更高频率） */
void Timer_ISR(void) {
    ulRunTimeCounter++;
}

/* FreeRTOSConfig.h 中定义 */
#define portALT_GET_RUN_TIME_COUNTER_VALUE(x)  \
    do { x = ulRunTimeCounter; } while(0)

Step 3：添加串口命令处理代码

在串口接收任务或命令处理函数中，添加以下代码：

/* ===== 串口命令处理函数（在你的 uart.c / serial_cmd.c 中） ===== */

#include "FreeRTOS.h"
#include "task.h"

/* 你的串口输出函数 */
extern void Serial_Print(const char *str);

void ProcessDebugCommand(const char *cmd)
{
    /* --- 命令: vTaskList — 获取任务状态快照 --- */
    if (strcmp(cmd, "vTaskList") == 0)
    {
        char *buf = pvPortMalloc(1024);
        if (buf)
        {
            vTaskList(buf);                    // FreeRTOS API
            Serial_Print(buf);
            vPortFree(buf);
        }
        return;
    }

    /* --- 命令: vTaskStats — 获取 CPU 占比 --- */
    if (strcmp(cmd, "vTaskStats") == 0)
    {
        char *buf = pvPortMalloc(1024);
        if (buf)
        {
            vTaskGetRunTimeStats(buf);        // FreeRTOS API
            Serial_Print(buf);
            vPortFree(buf);
        }
        return;
    }
}
提示：如果内存紧张，可以把 pvPortMalloc(1024) 改为静态分配的全局数组，避免动态内存碎片。

Step 4：发送命令验证

将固件烧录到开发板，打开串口工具连接后：

  1. 在发送区输入 vTaskList，点击发送
  2. 接收区应看到类似以下格式的数据：
Task           State   Priority  Stack    #
IDLE           R       0         117      1
uartTask       B       2         458      3
sensorTask     B       1         187      4
Tmr Svc        B       3         102      2
LEDTask        B       1         312      5
KeyScan        S       2         234      6
  1. 发送 vTaskStats，接收区应显示 CPU 占比：
Task           Abs Time    % Time
IDLE           5123456     65%
uartTask       1234567     16%
sensorTask     987654      12%
Tmr Svc        345678      5%
LEDTask        123456      2%
  1. 切换到 RTOS 监控页面，数据自动解析并显示

准备工作

在 FreeRTOS 项目中添加以下代码到调试命令处理函数：

命令代码用途
vTaskListvTaskList(buffer); Serial_Print(buffer);获取任务状态快照
vTaskStatsvTaskGetRunTimeStats(buffer); Serial_Print(buffer);获取 CPU 运行时间占比
注意：
- 使用 vTaskGetRunTimeStats() 前需要配置 configGENERATE_RUN_TIME_STATS 和 portALT_GET_RUN_TIME_COUNTER_VALUE
- vTaskList() 需要 configUSE_TRACE_FACILITY 为 1

使用步骤

步骤 1：连接串口
在串口工具中打开串口连接目标板。
步骤 2：发送命令
在串口工具中发送 vTaskList 或 vTaskStats 命令。
步骤 3：自动解析
切换到 RTOS 监控页面，数据自动解析显示（默认自动检测格式）。
步骤 4（可选）配置总栈
点击 配置总栈 为每个任务录入总栈大小，即可看到栈使用百分比进度条。

表格列说明

列说明颜色含义
#FreeRTOS 内部任务编号-

状态就绪 / 阻塞 / 挂起 / 删除行背景随状态着色
优先级任务优先级（0 = IDLE 最低）-
栈余量(字)剩余栈空间（单位：字）-
栈使用%进度条（需先配置总栈）绿<60% 橙60-85% 红>85%
CPU%CPU 占用百分比来自 vTaskGetRunTimeStats

工具栏操作

  • 格式选择：自动检测 / 强制 vTaskList / 强制 vTaskGetRunTimeStats
  • 暂停 / 继续 - 冻结/恢复实时更新
  • 清空 - 清除所有数据和行缓存
  • 立即解析 - 手动触发解析（支持从剪贴板粘贴数据）
  • 配置总栈 - 录入各任务总栈大小，自动计算栈使用百分比
调试技巧：
- 检测栈溢出：栈使用% > 85% 时标红，说明该任务栈空间可能不足
- 观察任务行为：状态在"就绪"和"阻塞"间切换表示正常调度
- CPU 负载分析：IDLE 占比越低，系统负载越高

9. CAN 总线 NEW

CAN 总线页面 — 通过 CANable (slcan 协议) 适配器收发 CAN 帧。支持标准帧/扩展帧、周期发送、实时收发表格。

准备工作

将 CANable 设备通过 USB 连接到电脑，确保驱动已安装（CANable 在 Windows 上显示为虚拟串口）。

使用步骤

步骤 1：选择串口
在连接栏选择 CANable 对应的串口号。
步骤 2：设置 CAN 波特率
选择与 CAN 总线一致的波特率（常用 500 kbit/s）。
步骤 3：打开 CAN
点击 ⚡打开 CAN，工具自动配置 slcan 速率并开启通道。
步骤 4：发送 CAN 帧
在发送面板输入 ID(Hex)、DLC、数据(Hex)，点击 发送。

发送面板说明

控件说明
ID (Hex)CAN ID 十六进制输入（如 123、0x456）
标准帧 / 扩展帧标准帧 ID 范围 0x000~0x7FF，扩展帧 0x00000000~0x1FFFFFFF
DLC数据长度码，0~8
数据 (Hex)十六进制数据字节，空格分隔（如 01 02 03 04）
周期发送勾选后按设定间隔自动循环发送

接收表格

接收到的 CAN 帧以表格形式显示：

  • # — 连续帧序号
  • 时间 — 精确到毫秒的时间戳
  • 方向 — TX(蓝色) / RX(绿色)
  • ID — CAN ID（十六进制）
  • DLC — 数据长度
  • 数据 — 十六进制字节

10. 数据日志录制 & 回放 NEW

数据录制 & 回放是全局能力，无论你在哪个页面，都可以录制 DataBus 上的所有数据，之后离线回放查看。

录制

录制、保存、回放按钮位于左侧侧边栏底部。

步骤 1：点击绿色三角录制按钮
按钮变为红色方块，开始录制所有数据通道。
步骤 2：正常操作
切换页面、收发数据、打开关闭串口——所有操作均被记录。
步骤 3：停止录制
再次点击红色方块停止。日志进入待保存状态，不会自动保存为最终文件。
步骤 4：保存录制
点击旁边的保存按钮，选择 .estlog 保存位置。

文件格式

.estlog 格式（JSON Lines）：

{"ts":1683532800123,"ch":"transport.can.0","payload":"t123456780102030405060708"}
{"ts":1683532800156,"ch":"transport.serial.0","payload":"32352E330D0A"}
{"ts":1683532800189,"ch":"transport.can.0","payload":"t4567850102"}

回放

点击左侧底部的回放按钮打开回放控制栏，加载 .estlog 文件后可播放：

  • 📂 加载 — 选择 .estlog 文件
  • ▶ / ⏸ — 播放 / 暂停
  • ⏹ — 停止回放
  • 进度条 — 拖拽跳转到任意位置
  • 1.0x — 循环切换变速 0.5x / 1x / 2x / 5x / 10x

回放时，所有页面（波形图、协议解析、RTOS 监控、CAN 总线）如同连接真实设备一样自动接收数据。

典型场景：
- 现场抓到一个异常 → 点录制 → 回来慢慢分析
- 同事分享 .estlog 文件 → 导入回放，所有页面联动
- 调整协议解析模板 → 回放同一段数据对比效果
